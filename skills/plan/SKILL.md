# Plan - TDD 구현 계획 작성

요구사항: **$ARGUMENTS**

아래 순서로 실행하세요.

## 1단계: 코드베이스 파악

현재 프로젝트 구조를 파악합니다:
- .sln, .csproj 파일 위치 확인
- 관련 기존 코드 검색 (있다면)

## 2단계: 계획 작성

다음 형식으로 구현 계획을 작성하세요:

### 목표
(한 줄 요약)

### 테스트 시나리오 (Red 단계)
TDD 원칙에 따라 먼저 작성할 테스트 목록:
- [ ] 테스트1: `메서드명_상황_기대결과` 형식
- [ ] 테스트2: ...
- [ ] 엣지 케이스: ...

### 구현 단계 (Green 단계)
1. (최소 구현 - 테스트 통과만을 위한)
2. ...

### 리팩토링 포인트 (Refactor 단계)
- 중복 제거 예상 부분
- 네이밍 개선 예상 부분

### 영향 범위
- 새로 만들 파일:
- 수정할 파일:
- 건드리지 않을 파일:
