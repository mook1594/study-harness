# Review - 코드 리뷰

$ARGUMENTS

아래 순서로 리뷰를 진행하세요.

## 1단계: 변경사항 파악

```bash
git diff HEAD~1
```

또는 $ARGUMENTS에 파일 경로가 있으면 해당 파일을 읽습니다.

## 2단계: 리뷰 체크리스트

각 항목을 확인하고 결과를 작성하세요:

### TDD 준수
- [ ] 테스트가 구현보다 먼저 작성되었는가? (커밋 순서 확인)
- [ ] 모든 공개 메서드에 테스트가 있는가?
- [ ] 테스트명이 `메서드_상황_기대결과` 형식인가?

### 코드 품질
- [ ] 메서드가 한 가지 일만 하는가? (20줄 초과 시 분리 권장)
- [ ] 중복 코드가 없는가?
- [ ] 매직 넘버/문자열이 없는가? (상수로 추출)

### C# 컨벤션
- [ ] 클래스/메서드: PascalCase
- [ ] 변수/파라미터: camelCase
- [ ] 불필요한 using 없는가?

### 설계
- [ ] 인터페이스가 실제로 필요한가? (구현체 1개면 불필요)
- [ ] 예외처리가 적절한가?

## 3단계: 결과 출력

형식:
```
✅ 통과: (항목)
⚠️ 개선 권장: (항목) → 제안
❌ 수정 필요: (항목) → 이유
```

수정이 필요한 항목이 있으면 직접 수정할지 물어보세요.
